## 技术栈与运行环境
- 前端首期采用静态页面 + Nginx 容器（快速落地 UI），后续可平滑升级到 React/Vite。
- 浏览器兼容：现代浏览器（Chrome/Edge/Firefox/Safari 最新版）。
- Docker：在现有 compose 中新增 `web` 服务，映射 `./frontend` 到容器 `/usr/share/nginx/html`，端口 `3000`。

## 页面与交互需求
- 登录页（效果同图）：
  - 字段：学号、密码；显示系统名与分团名；“登录”按钮；“立即注册/忘记密码”链接。
  - 行为：校验非空；后续对接后端返回错误时提示。
- 注册页（效果同图）：
  - 字段：姓名、学号、密码、确认密码、手机号、学院（下拉）、性别（男/女）、生日（年/月/日）、入团年份（2014–当前年下拉）、头像上传。
  - 行为：基础前端校验（非空、手机格式、密码一致、图片大小类型）；后续对接后端扩展校验与错误提示。

## UI 设计实现
- 背景：线性渐变 + 居中白色卡片，圆角阴影。
- 表单：统一输入框高度、间距与标签样式；按钮主色为蓝色；错误信息采用红色小字显示。
- 响应式：卡片在移动端宽度自适应，最小 320px，最大 400–480px。

## 文件结构（新增）
- `frontend/`
  - `login.html`：登录页
  - `register.html`：注册页
  - `styles.css`：通用样式（背景、卡片、表单控件、按钮）
  - `script.js`：基础校验与交互占位（如显示/隐藏密码、校验必填、提示信息）
- Docker：在 compose 中添加 `web` 服务（Nginx）。

## 表单校验与数据模型
- 必填校验：所有标注 * 的字段非空。
- 密码：长度 ≥ 8（可包含数字/字母/特殊字符），注册时需与确认密码一致。
- 手机号：大陆手机号格式（正则占位）。
- 性别：枚举男/女。
- 入团年份：范围 2014–当前年；生日选择年/月/日。
- 头像：大小限制（如 ≤ 2MB），类型 `image/jpeg/png/webp`。

## 后端接口约定（占位，便于后续接入）
- `POST /auth/login`：`{ student_no, password }` → `200` 返回会话（后续使用 HttpOnly Cookie）。
- `POST /auth/register`：提交注册表单完整信息 → `201` 创建用户（头像文件采用 multipart/form-data）。
- 成功/失败提示：统一 JSON 格式 `{ code, message }`，前端显示 message。

## 交付与验证
- 在 Docker 中启动前端服务，访问：`http://localhost:3000/login.html` 与 `http://localhost:3000/register.html`。
- 视觉对齐：字段布局顺序、标题与辅助文案与提供图片一致；间距与按钮样式遵循示例。
- 验证：本地必填与基本格式校验可用；后端未接入阶段不真实提交（或采用假接口）。
